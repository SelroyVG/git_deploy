Параметры запуска:
  Обязательные:
  "-l <link>" -- ссылка на репозиторий
  "-d <directory>" -- рабочая директория для репозитория
  
  Необязательные: 
  "--include-patterns" -- если параметр указан, после того как репозиторий будет развёрнут, пользователю будет предложено настроить текстовые файлы.
  "--disable-input" -- если параметр указан, пользователю не будет предлагаться ввод с клавиатуры.
  "-b <name>" -- название используемой ветки (по умолчанию master)
  "-h", "--help" -- отобразить помощь

Подстановка значений:
  Программа ищет два файла:
    1) Файл, в котором записано в файлах с каким расширением нужно совершить подстановку (по умолчанию config/extensions).
    2) Файл в формате CSV в котором записаны правила подстановок (по умолчанию config/sliza_templates.csv).
  Если какой-то из файлов не найден, программа попросит ввести пользователя название файла или полный путь к нему в ручную.
  
CSV-файл должен быть составлен по следующим правилам:
  1) Каждая строка -- одно правило замены.
  2) Правило замены записывается в таком виде:
  
  <Шаблон>,<Подставляемое_значение>

Задача:

Разработать программу, позволяющую автоматически развернуть git-репозиторий (не вдаваясь в команды git-а) в указанной папке с возможностью опционального поиска и замены подстрок в текстовых файлах.

Весь код программы разделён на две части:

git_deploy.cpp - содержит основной код программы, разворачивающий репозиторий.
sliza_deploy.cpp (надо не забыть переименовать) - содержит вспомогательные функции, производящие в скачанном репозитории поиск и замену подстрок.

Алгоритм программы:

Вначале создаётся папка .temp/temp_rep, в ней будет временно храниться скачанный репозиторий. После этого начинается проверка наличия необходимых входных параметров. Если чего-то не хватает, работа программы будет завершена с ошибкой. 

После того как все приготовления закончены, программа скачивает указанный репозиторий при помощи команды "git clone" в папку .temp/temp_rep. 

ОСТОРОЖНО! Если предполагается что программа должна запускаться в автоматическом режиме, убедитесь что для скачивания не потребуется логин и пароль от репозитория. Например, для github можно создать пару ключей SSH и поместить открытый в вашем профиле.

После того как репозиторий скачан, если присутствует параметр --include-patterns, задействуются функции из файла sliza_deploy.cpp. Программа берёт левого столбца файла config/sliza_templates.csv подстроки, парсит скачанные файлы и заменяет эти подстроки на те, что указаны в правом столбце. Стоит учесть, что парситься будет не всё подряд, а только файлы с расширениями указанными в файле config/extensions (точнее даже будет сказать что будут парситься файлы, названия которых оканчиваются одной из строк, указанных в файле).

Затем, репозиторий сохраняется в ту папку, которую указал пользователь. Если этой папки не существует, программа создаст её сама. Затем, вне зависимости от содержимого этой папки, репозиторий будет скопирован туда с заменой.
